## 1. 系统概述

### 1.1 方案对比

| 版本 | 雷达数量 | 总成本 | 接线复杂度 | 说明 |
|:----:|:--------:|:------:|:----------:|------|
| V1.0 | **2个** | ¥344 | 中等 | 左右各一个雷达，各占一个UART |
| **V2.1** | **1个** | **¥261** | **简单** | **居中安装，angle字段判左右** |

**为什么1个雷达就够了？**

MS60-3015 的 BSD 协议为每个检测到的目标输出 **角度值（angle）**，范围 -64°~+63°，左负右正。雷达水平 FOV 为 **±40°**，覆盖 80° 范围：

```
雷达居中安装, 天线正对正后方：

    左后方          正后方          右后方
  ─40° ─── ─15° ─── 0° ─── +15° ─── +40°
         │                      │
   盲区范围                 盲区范围
   (左车道)                (右车道)
```

- angle < 0 → 目标在左后方
- angle > 0 → 目标在右后方
- 按 angle 范围即可区分左右盲区，无需两个雷达

### 1.1.1 AT6010 协议实测验证 (2026-06-16)

MS60-3015 基于**隔空 AT6010 60GHz SOC**，协议遵循 AT6010 HCI Protocol v1.4。

**主动上报帧格式（雷达→主机）：**
```
HEAD(0x5A) + LEN(1B) + PAYLOAD(LEN bytes) + CHECK(1B)
```
- CHECK = sum(HEAD + LEN + PAYLOAD) & 0xFF（1 字节校验和）
- PAYLOAD 首字节为 TYPE（上报类型）

**实测 BSD 帧（TYPE=0x07，厂商自定义）：**
```
5A 09 07 01 00 00 00 03 2D 00 00 9B
│  │  │  │  └─────┬─────┘ └┬┘ └┬┘  │
│  │  │  │    reserved   status angle reserved
│  │  │  └─ obj_count=1           (0x03=运动)
│  │  └─ TYPE=0x07 (BSD vendor)   (0x04=微动)
│  └─ LEN=9 bytes
└─ HEAD=0x5A
```
- 角度(angle): `int8_t`, 符号角度，正=右，负=左
- 状态(status): `0x03`=运动目标, `0x04`=微动目标
- **仅检测到目标时上报，无目标则静默**

**命令帧格式（主机→雷达）：**
```
HEAD(0x58) + CMD_GROUP(1B) + CMD(1B) + LEN(1B) + PARAMS(LEN) + CHECK(1B)
```
- 配置命令组 CMD_GROUP = 0x01
- 命令示例：ENABLE_BSD(0x05), SET_DET_RANGE(0x02), SET_SENSITIVITY(0x03)

**关键参数：**
- 默认波特率：**921600**（不是 115200）
- 供电：**支持 5V**（文档标注已支持）
- 启动日志：`at6010 boot v1.9\r\nNAK\r\ncache on\r\njump to run 0x0`
- ESP32 引脚：GPIO16(RX2) ← 雷达TX，GPIO17(TX2) → 雷达RX

### 1.4 三大输出设备 — 转向灯 / LED指示灯 / 蜂鸣器

系统包含三类独立的输出设备，各司其职：

| 设备 | 硬件 | 用途 | 感知对象 |
|------|------|------|----------|
| **转向灯** | 12V LED（前后四灯），MOSFET 驱动 | 向**其他交通参与者**发出转向/双闪/警示信号 | 外部车辆/行人 |
| **LED指示灯** | 5mm 红色 LED ×2，GPIO 直驱 | 向**骑行者**发出后方危险方向提示 | 骑行者本人 |
| **蜂鸣器** | 压电蜂鸣器，PWM 驱动 | 向**骑行者**发出听觉警报 | 骑行者本人 |

```
               ┌──────────┐
               │  ESP32   │
               └────┬─────┘
                    │
      ┌─────────────┼─────────────┐
      │             │             │
  ┌───▼───┐   ┌────▼────┐   ┌───▼───┐
  │MOSFET │   │  GPIO   │   │ GPIO  │
  │×4 驱动│   │ 直驱LED │   │ 蜂鸣器│
  └───┬───┘   └────┬────┘   └───┬───┘
      │             │             │
  12V转向灯    指示灯(L/R)   5V蜂鸣器
  前左/前右    GPIO25/26     GPIO12
  后左/后右    5mm红色       压电式
```

#### 1.4.1 转向灯（Turn Signals）

转向灯是系统的**主要外部信号**，安装于车辆前后四角，12V 摩托车 LED 成品，经 MOSFET 由 ESP32 控制。

| 场景 | 转向灯行为 | 频率 | 目的 |
|------|-----------|:---:|------|
| **左转向** | 左前 + 左后同步闪烁 | 1.5Hz | 告知后方你即将左转 |
| **右转向** | 右前 + 右后同步闪烁 | 1.5Hz | 告知后方你即将右转 |
| **双闪 (HAZARD)** | 四灯同步闪烁 | 3Hz (快闪) | 紧急停靠 / 恶劣天气 |

**关键设计**：转向辅助检测到危险时**不改变转向灯频率**。转向灯始终保持标准 1.5Hz，让其他道路使用者正确理解你的转向意图。危险提醒通过指示灯和蜂鸣器单独传达给骑行者。

#### 1.4.2 LED 指示灯（Indicator LEDs）

指示灯是**骑行者专属视觉提醒**，安装在手把仪表区域，5mm 红色高亮 LED 串 220Ω 电阻，ESP32 GPIO 直驱（无需 MOSFET）。自然对应左右方向，骑行时余光可见。

| 场景 | 左指示灯 (GPIO25) | 右指示灯 (GPIO26) | 目的 |
|------|:---:|:---:|------|
| **无目标** | 灭 | 灭 | 安静巡航 |
| **后方低速目标** (2-3 m/s) | 🔴 1Hz 慢闪 | — | 提醒侧方有车 |
| **后方高速目标** (≥3 m/s) | 🔴 4Hz 快闪 | — | 后方快速接近 |
| **转向辅助** | 🔴 常亮 | — | 警告：转向方向有车 |
| **转向辅助+高速目标** | 🔴 4Hz 快闪 | — | 高速目标覆盖转向（更高优先） |

> V2.6: BSD+RCW合并为统一后方监测，按速度两级报警。低速→IND_MODE_BSD(1Hz)，高速→IND_MODE_RCW(4Hz)。

**优先级规则**（仅适用后方监测 vs 转向辅助）：
- 后方监测检测到快速目标时，指示灯由 RCW 模式控制（4Hz 快闪）。
- 转向辅助检测到危险时，指示灯常亮（被 RCW 快闪覆盖时降级）。

#### 1.4.3 蜂鸣器（Buzzer）

蜂鸣器是**听觉警报**，12V 高分贝有源蜂鸣器，通过 **IRLZ44N MOSFET (Q1)** 驱动。GPIO12 经 1kΩ 电阻接栅极，漏极接蜂鸣器负极，蜂鸣器正极接 12V。HIGH=响，LOW=灭。

| 优先级 | buzzer_mode | 触发条件 | 蜂鸣模式 |
|:---:|:---:|------|------|
| **最高 (2)** | 2 | RCW 后碰撞预警 | 4Hz 短促鸣响 |
| **高 (3)** | 3 | 转向辅助检测到同侧危险 | **持续长鸣** |
| **中 (1)** | 1 | BSD 盲区首次检测到目标 | 单次 100ms 短鸣，冷却 5s |
| **最低 (0)** | 0 | 无警报条件 | 静音 |

**优先级规则**：
- `buzzer_mode == 2`（高速4Hz）/ `3`（转向长鸣）覆盖模式 1（低速单鸣）。
- 低速蜂鸣有 5 秒冷却（由 `sys.bsd_beep_cooldown` 控制）。

#### 1.4.4 场景交互矩阵

| # | 场景 | 转向灯 | 左指示灯 | 右指示灯 | 蜂鸣器 |
|---|------|:---:|:---:|:---:|:---:|
| 1 | 无操作，后方无车 | 灭 | 灭 | 灭 | 静音 |
| 2 | 无操作，后方低速目标 | 灭 | 🔴 1Hz慢闪 | 灭 | 🔔 单鸣 |
| 3 | 无操作，后方高速目标 | 灭 | 🔴 4Hz快闪 | 灭 | 🔊 4Hz |
| 4 | 打左转向，左后安全 | ↙ 1.5Hz | 灭 | 灭 | 静音 |
| 5 | 打左转向，左后有车 | ↙ 1.5Hz | 🔴 常亮 | 灭 | 🔊 长鸣 |
| 6 | 打左转 + 后方高速目标 | ↙ 1.5Hz | 🔴 4Hz快闪 | 灭 | 🔊 4Hz |

### 1.5 工作模式流程

```
开机初始化
    │
    ├── 配置雷达 (UART2, 921600bps)
    │     ├── 设置探测距离 30m
    │     ├── 设置灵敏度
    │     └── 启用 BSD 主动上报 (TYPE=7, 4B目标格式)
    │
    ├── 主循环 (每 50ms)
    │     │
    │     ├── 读取转向开关状态 (自锁式)
    │     │     ├── 左位(LOW) → 左转向灯闪烁 (1.5Hz)
    │     │     ├── 右位(LOW) → 右转向灯闪烁 (1.5Hz)
    │     │     └── 中间(HIGH) → 关闭转向灯
    │     │
    │     ├── 解析雷达UART数据 (BSD主动上报, TYPE=7)
    │     │     ├── obj_num(2B LE) + reserved(2B)
    │     │     ├── 每个目标4B: range(s8,m) + angle(s8,°) + velo(s8,m/s) + objId(s8)
    │     │     └── 按 angle 分类左/右 (REAR_ANGLE_IS_LEFT/RIGHT)
    │     │
    │     ├── 后方监测逻辑 (BSD+RCW合并, updateRCW)
    │     │     ├── 低速目标 (≥low_speed, <speed_threshold) → IND_MODE_BSD (1Hz慢闪) + 蜂鸣单声(5s冷却)
    │     │     ├── 高速目标 (≥speed_threshold) → IND_MODE_RCW (4Hz快闪) + 蜂鸣4Hz
    │     │     └── 超时解除 (hold_time)
    │     │
    │     ├── 转向辅助逻辑 (打转向灯时, 仅新帧触发)
    │     │     ├── 帧新鲜度检查 (避免旧帧重复触发 — V2.5修复)
    │     │     ├── 同侧有接近目标 (速度>7.2km/h, 距离<30m)
    │     │     │     ├── 设置同侧灯 IND_MODE_TURN (常亮)
    │     │     │     ├── 蜂鸣器 持续长鸣 (buzzer_mode=3)
    │     │     │     └── 转向灯保持正常 1.5Hz (不改变频率)
    │     │     └── 同侧无危险 → 模式归零, 蜂鸣停
    │     │
    │     ├── 指示灯集中状态机 (updateIndicatorLEDs)
    │     │     ├── 读取 ind_left_mode / ind_right_mode
    │     │     ├── IND_MODE_OFF → 灭
    │     │     ├── IND_MODE_BSD → 1Hz 慢闪 (500ms半周期)
    │     │     ├── IND_MODE_RCW → 4Hz 快闪 (125ms半周期, 与蜂鸣同步)
    │     │     └── IND_MODE_TURN → 常亮
    │
    └── 循环
```

|| 类型 | 安装方式 | 手感 | 参考价 ||
||------|----------|------|:------:||
|| **手把管夹式** | 夹在车管上(22mm管径) | 大拇指推拨 | ¥15~22 ||

### 2.4 关键器件详细说明

**MS60-3015 雷达（核心器件）：**
- 60GHz FMCW 毫米波，专为户外低速车辆 BSD 设计
- 探测距离：汽车50m / 摩托车30m / 自行车25m / 行人20m
- 直接 UART 输出 **角度(°) + 状态**，每目标 **7 字节**（含保留字段）
- 水平 FOV **±40°** — 单个居中即可覆盖两侧盲区
- 主动上报模式，100ms刷新 (10Hz)，无需轮询
- 3.3V / 14mA 超低功耗，尺寸 29.8×15mm
- 引脚：①3.3V ②GND ③OUT(悬空) ④RX ⑤TX
- 默认波特率 **921600**，固件使用 921600

**IRLZ44N MOSFET：**
- 逻辑电平 N-MOS，Vgs(th) ≈1V，ESP32 3.3V GPIO **完全可驱动**
- 导通电阻 Rds(on) ≈22mΩ @Vgs=5V（5V时更好，但3.3V也能用）
- 连续漏极电流 47A，远超转向灯 (<1A)
- TO-220 封装，不发热（0.8A × 22mΩ = 14mW 功耗）
- 每个 MOSFET 需配：1kΩ 栅极限流电阻 + 10kΩ 栅-源下拉电阻

**XL7015 降压模块：**
- 输入 5~80V DC（48V电瓶充满55V也在安全范围内）
- 输出 12V 5A，效率 ~85%
- 模块自带散热片、电感、滤波电容
- 输出 12V 母线同时给：LED 转向灯×4 (≤40W) + AMS1117 → ESP32/雷达

**AMS1117-3.3V 稳压模块：**
- 输入 12V，输出 3.3V 1A
- 给 ESP32 + MS60-3015 雷达供电
- **重要**：雷达的 3.3V 必须从这个 AMS1117 取电，不要用 ESP32 板载 3.3V（板载电流不够 50mA+，且会引入噪声干扰雷达）

### 4.2.1 看门狗

// 雷达供电: 5V (实测4.7V, 文档确认支持5V)

### 4.2.2 蜂鸣器优先级

蜂鸣器采用状态机控制，优先级从高到低：

| 优先级 | buzzer_mode | 触发条件 | 蜂鸣模式 |
|:------:|:-----------:|----------|----------|
| 最高 | 2 | RCW 后碰撞预警 (>10.8km/h) | 4Hz 急促鸣响 |
| 高 | 3 | 转向辅助检测到同侧危险 | 持续长鸣 |
| 中 | 1 | BSD 盲区首次检测到目标 | 单次短鸣 (100ms)，冷却 5s |
| 最低 | 0 | 无警报 | 静音 |

**规则**：低优先级不会打断高优先级。BSD 只在 `buzzer_mode==0` 时设置为 1；RCW 设置为 2（覆盖 BSD）；转向辅助设置为 3（覆盖 BSD，但不覆盖 RCW）。关转向时清除 mode 2/3，关 RCW 时清除 mode 2。

### 4.3 角度分区策略

```
雷达法线方向 = 正后方 (0°)

左盲区   [-40° ~ -15°]   → bsd_l_active = true
           (左侧车道后方来车)

正后方    [-14° ~ +14°]  → 不触发盲区报警
           (同车道后方跟车, 正常)

右盲区   [+15° ~ +40°]   → bsd_r_active = true
           (右侧车道后方来车)

转向辅助范围 (比 BSD 稍宽, 为安全):
  左转辅助: [-40° ~ -5°]
  右转辅助: [+5° ~ +40°]
```

### 4.4 核心算法

```cpp
// === BSD 盲区：按 angle 分类左右 ===
void updateBSD() {
    for each target:
        if angle in [-40°, -15°] AND range < 30m:
            bsd_l_active = true
        if angle in [+15°, +40°] AND range < 30m:
            bsd_r_active = true
        // 正后方 [-14°, +14°] 不触发
    if !rcw_l_active: INDICATOR_L = bsd_l_active ? ON : OFF
    if !rcw_r_active: INDICATOR_R = bsd_r_active ? ON : OFF
    if bsd_*_active && !warned && buzzer_mode == 0:
        buzzer_mode = 1   // 单次短鸣, 5s冷却
}

// === RCW 后碰撞预警：高速接近 ===
void updateRCW() {
    for each target:
        if abs(velocity) < 3m/s OR range > 25m: skip
        if angle in [-40°, -5°]:  leftFast = true
        if angle in [+5°, +40°]:  rightFast = true
    if leftFast:  INDICATOR_L = ON; buzzer_mode = 2  // 4Hz
    if rightFast: INDICATOR_R = ON; buzzer_mode = 2
}

// === 转向辅助：打灯时判断同侧危险 (仅新帧触发) ===
function turn_assist(turn_dir):
    if frame_timestamp == last_ts: return  // V2.5: 防旧帧重复触发
    for each target:
        if velocity < 2m/s OR range > 30m: skip
        if turn_dir == LEFT  AND angle in [-40°, -5°]: danger = true
        if turn_dir == RIGHT AND angle in [+5°, +40°]: danger = true
    if danger:
        set_indicator(on, side)  // 同侧指示灯常亮
        buzzer_mode = 3          // V2.5: 持续长鸣
    else:
        clear_indicator(side)    // 保留 RCW 的判断
        clear_buzzer_if_not_rcw()
```

### 4.5 主循环时序

```
loop() 每 50ms 执行一次:

  [0ms]   读取转向开关 (自锁式, 直读电平)
  [0ms]   雷达 UART 数据读取 + 帧同步解析
  [2ms]   BSD 盲区逻辑 (遍历目标, angle 分类)
  [4ms]   RCW 后碰撞预警 (高速目标筛选)
  [6ms]   转向辅助逻辑 (如打灯中, 同侧危险判断)
  [8ms]   LED 闪烁状态机 (每 667ms 翻转 = 1.5Hz)
  [10ms]  蜂鸣器状态机
  [12ms]  串口调试输出 (每 500ms)
  [50ms]  循环延时
```

**雷达安装要求（严格，否则角度判不准）：**
- ✅ **居中安装**，在车尾部（后挡泥板或货架）正中央
- ✅ 天线面朝 **正后方**，与车体纵轴平行（0° 偏差）
- ✅ 安装高度 **0.5~1.5m**（电瓶车推荐 0.7~0.9m）
- ✅ 前方（天线面朝向的反方向）**绝对净空**，无金属遮挡
- ❌ 不能歪斜安装——角度偏移会导致左/右分区漂移
- ❌ 雷达天线前不能有金属牌照架、车筐等

> **为什么必须居中朝正后方？**
>
> 因为 angle 值的参考系是雷达法线。只有雷达法线 = 车体纵轴时，
> angle < 0 才准确对应车辆左侧，angle > 0 对应右侧。
> 如果雷达偏左或偏右安装，角度分区会发生系统性偏移。

**转向灯安装：**
- 前转向灯：安装在车把左右两侧或前叉上，用自带金属支架 + M6 螺丝固定
- 后转向灯：安装在坐垫后方的车架两侧
- 转向灯线头引出后接硅胶延长线，接头务必防水（热缩管+硅胶密封）

### 5.2 电路盒安装

- 防水盒固定在车篮下方或座桶内
- ESP32 + XL7015 + AMS1117 + MOSFET + 蜂鸣器均置于盒内
- 盒体侧面开孔穿线，出线口用防水接头
- 雷达模块建议外置于车尾货架处（天线不能有金属遮挡）

### 5.3 电气安全注意事项

1. **48V 高压务必先接保险丝**（5A 慢熔保险，紧贴电瓶正极）
2. **XL7015 输入电容** 220μF/63V，输出电容 100μF/25V（模块自带则不用外加）
3. **所有大功率地线星型汇接**，避免回路干扰
4. **UART 线材**使用屏蔽线或用双绞线，远离 48V 电源线
5. **转向灯负极**接 MOSFET 漏极，不是直接接地
6. **雷达 3.3V 供电**单独从 AMS1117 取电，不要从 ESP32 板载 3.3V 取电
7. **焊接完成先目测检查**，再用万用表测通断和短路，最后才通电

## 8.5 钥匙开关控制 (3.3V信号 → 48V电源通断)

电瓶车钥匙开关输出的 **3.3V 信号**，通过 P-MOSFET 高边开关控制 BSD 系统 48V 主电源。

### 电路

```
                 48V+ ──→ IRF9540 S ── D ──→ XL7015输入 (BSD系统)
电瓶48V+ ────────────────┘         │
                                   G ──┬── R1(47kΩ) ── S8050 C
                                   │   │              E ── GND
                                   │   │              B ── R3(10kΩ) ── 钥匙3.3V
                                   │   │
                                   │   └── R2(10kΩ) ── 48V+ (S极)
                                   │
电瓶48V- ───────────────────────────┴── GND
```

### 工作原理

| 钥匙状态 | S8050 | IRF9540 G极电压 | Vgs | MOSFET |
|:--|:--|--:|--:|:--:|
| OFF (0V) | 截止 | 48V (R2上拉) | 0V | **关断** |
| ON (3.3V) | 导通 | 39.6V (R1/R2分压) | **-8.4V** | **导通** |

3.3V 控制 48V——小信号控大电流。钥匙电流仅 0.26mA，不增加原车电路负担。

**Vgs 恒在 -9.5V~-7.4V（电池满电 54V~亏电 42V），远低于 IRF9540N 的 ±20V 极限。**

### 元器件

| 元件 | 型号 | 封装 | 数量 | 淘宝 |
|------|------|------|:--:|------|
| P-MOSFET | **IRF9540N** | TO-220 | 1 | ¥2 |
| NPN 三极管 | **S8050** | TO-92 | 1 | ¥0.5 |
| 电阻 | **47kΩ 1/4W** | AXIAL-0.3 | 1 | ¥0.1 |
| 电阻 | **10kΩ 1/4W** | AXIAL-0.3 | 2 | ¥0.2 |

**总成本: ¥3。** 不需要继电器，不需要额外降压模块。

### 接线步骤

1. 找到钥匙 ON 后有 3.3V 的那根线
2. 3.3V → 10kΩ(R3) → S8050 B
3. S8050 E → GND
4. S8050 C → 47kΩ(R1) → IRF9540 G
5. IRF9540 G → 10kΩ(R2) → IRF9540 S (也是 48V+)
6. IRF9540 S → 电瓶 48V+
7. IRF9540 D → BSD 系统输入
